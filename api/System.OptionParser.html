Parses options from the string arguments.

System.String[] args

The string arguments.
System.String keyPrefix

The prefix for the arugment key. The default is "--", so an argument key-value pair specification can be written in format --key value (two argument slots). The argument value can be later retrieved by GetArg<T>(String)method. Flags without arugment values can be set using --flag --key value (the first argument is a flag named "flag", and the second argument is a key-value pair), where the existence of the flag can be determined later by HasKey(String) method.
System.String keySurfix

The surfix for the argument key. The default is ":", so an argument key-value pair specification can be written in format "key=value" (one argument slot). A flag can be set using falg="".
System.Boolean enableConsole

true if unspecified required argument

Type Description
OptionSet

An OptionSet storing the parsed arguments.
Remarks

Three styles of argument specification are supported. Different styles can be mixed.

  • --key valueOccupies two argument slots. The '--' key prefix can be replaced by a customized prefix specified by keyPrefix. The value can be retrieved by GetArg<T>(String)method from the returned OptionSet object. To specify a flag, simply skip the value. For example, cmd --flag1 --flag2 --arg value specifies two flags 'flag1' and 'flag2' and one argument with key 'arg'.
  • key=valueOccupies one argument slot. The '=' key surfix can be replaced by a customized prefix specified by keySurfix. The value can be retrieved by GetArg<T>(String)method from the returned OptionSet object. To specify a flag, make the value empty. For example, cmd flag1= flag2= --arg value specifies two flags 'flag1' and 'flag2' and one argument with key 'arg'. This is also an exmaple that two styles of argument specification can be mixed.
  • unkeyed argumentUnkeyed arguments are matched with unspecified required arguments in order. For example, suppose we define two required arguments "arg1", "arg2" and "arg3", then cmd --arg2 value2 value1 has one keyed argument "arg2" with its value being "value2"; and "value1" is an unkeyed argument that will be assigned to "arg1", because "arg1" is the first unspecified argument.
