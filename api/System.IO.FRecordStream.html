Class FRecordStream

A forward-only stream that allows storing records of indefinite size on the same base stream. Overhead is remarkably lower than a similar class RecordStream.

Syntax
public class FRecordStream : Stream, IDisposable, IBasicRecord
Remarks

This class allocate blocks of variable length to store data and works much like a forward-only linked list. In this sense backward-seeking is not supported by this class. A similar class RecordStream always allocates blocks of the same length and is able to seek back and forth. However, two methods SaveStatus and RestoreStatus are provided to alleviate this drawback. Also note that the memory usage of each instance of this class is fixed, with contrast to RecordStream which uses more memory when new blocks are created.

Examples

The following codes shows how SaveStatus and RestoreStatus are used to remember and go back to history position.

    var ms = new MemoryStream();
    var fr = new FRecordStream(ms);
    fr.LoadInfo();
    //fr.SaveStatus(); //Invalid operation! This record stream is new and right after LoadInfo it is still empty.
    fr.WriteString("abcdefg");
    fr.SaveStatus();
    fr.WriteString("hijklmn");
    fr.SaveStatus();
    fr.WriteString("opqrst");
    fr.Reset();
    Console.WriteLine(fr.ReadString()); //"abcdefg"
    Console.WriteLine(fr.ReadString()); //"hijklmn"
    Console.WriteLine(fr.ReadString()); //"opqrst"
    fr.RestoreStatus();
    Console.WriteLine(fr.ReadString()); //"opqrst"
    fr.RestoreStatus();
    Console.WriteLine(fr.ReadString()); //"hijklmn"
